# Lyric Video Studio 产品需求文档

## 1. 产品目标

Lyric Video Studio 是一个面向音乐创作者的 AI 歌词视频制作工具。目标不是做一个通用视频编辑器，而是复刻 LyricEdits 这类产品的核心体验：用户上传歌曲后，系统自动识别歌词、拆分分镜、生成静态视觉、叠加歌词字幕，并导出可发布的 MP4。

第一阶段优先跑通 MVP 闭环：

上传音频 -> 识别/编辑歌词 -> 生成分镜 -> 生成静态图片 -> 配置字幕 -> 合成 MP4 -> 下载发布。

## 2. 用户画像

- 独立音乐人：需要低成本快速制作 YouTube、TikTok、Reels 歌词视频。
- 乐队/厂牌运营：需要批量为歌曲制作宣传素材，关注效率和一致性。
- 内容创作者/剪辑师：希望先生成可用视频，再进入 Premiere、Final Cut、DaVinci Resolve 做高级精修。
- 非专业剪辑用户：不熟悉视频软件，但能理解歌词、画面、字幕样式和导出。

## 3. 核心用户动线

1. 音频输入与初审
   用户上传 MP3、WAV、M4A 等音频，创建项目。系统保存音频并准备转写。

2. 歌词识别与校对
   系统通过 ASR 或多模态模型生成带时间戳的歌词行。用户可以手动粘贴歌词，也可以编辑识别结果。

3. 全局风格定义
   用户配置 Story Prompt、Palette、Aspect Ratio、Art Style、Resolution，为后续分镜和图片生成提供统一方向。

4. 分镜生成与微调
   系统根据歌词和全局风格生成 scenes。用户逐个检查分镜 prompt、关联歌词和时间范围。

5. 静态视觉生成
   用户可以为单个 scene 或全部 scenes 生成图片。系统轮询任务状态，并把成功图片写回 scene。

6. 字幕包装
   用户选择字体、字号、位置、颜色、阴影和歌词动效。第一阶段导出重点支持静态/基础字幕样式。

7. 导出与下载
   系统使用 FFmpeg 合成音频、静态图片、字幕，生成 MP4。用户在线播放、打开或下载视频。

## 4. MVP 范围

MVP 必须包含：

- 音频上传与项目创建。
- 项目列表、项目详情和状态展示。
- 自动转写或手动歌词输入。
- 歌词编辑与基础时间戳保存。
- 全局风格参数保存。
- LLM/启发式分镜生成。
- scene prompt 编辑。
- 单图/批量图片生成与状态刷新。
- 字幕样式基础配置。
- 静态图 + 音频 + 字幕的 MP4 导出。
- 完成后可在线播放和打开 MP4。

## 5. 暂缓范围

以下能力放到 v2/v3，不进入第一轮闭环：

- 图片转视频的 Motion Prompt 渲染。
- AI Inpainting 局部擦除。
- 复杂角色一致性模型训练或参考图工作流。
- Premiere、Final Cut、DaVinci Resolve 工程文件导出。
- 多轨音频、逐词 karaoke 高亮和高级节拍分析。
- 团队协作、版本历史、评论批注。

## 6. 当前项目现状

代码库已经具备一部分基础能力：

- `src/modules/lyric-videos/service.ts` 已包含项目创建、歌词替换、分镜生成、图片生成、图片状态同步和 FFmpeg 静态导出逻辑。
- `/api/lyric-videos` 路由体系已经存在，覆盖项目、歌词、分镜、图片和导出。
- `src/config/db/schema.ts` 及三套数据库模板已经包含 `lyric_video_project`、`lyric_video_line`、`lyric_video_scene`、`lyric_video_export`。
- `src/app/[locale]/dashboard/lyric-videos/page.tsx` 已有基础 dashboard 页面，可创建项目并走部分流程。
- `docs/research/lyricedits.ai/` 已有目标站的页面结构、行为和视觉研究材料。

主要缺口：

- 全局风格配置 UI 还不完整。
- 前端 timeline 仍是基础 tabs，不是专业编辑器式工作台。
- 歌词时间戳编辑能力偏弱。
- scene split、scene 时间范围编辑和歌词关联维护尚未完成。
- 字幕样式配置还没有完整前端面板。
- 角色一致性目前只能通过 prompt 约束，没有独立角色锁定体验。

## 7. 成功标准

- 用户能在 dashboard 内完成一次完整流程：上传音频 -> 生成/保存歌词 -> 生成分镜 -> 生成图片 -> 导出 MP4。
- 导出的 MP4 包含原始音频、分镜图片和同步歌词字幕。
- 失败状态能清晰展示，用户可以重试图片生成或导出。
- 所有新增功能通过 `pnpm build`。
- 文档 checklist 能持续反映真实进度，每完成一个功能就打勾。
