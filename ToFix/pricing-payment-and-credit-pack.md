# Pricing / Payment TODO

> 这个文档记录价格页、订阅、一次性积分包和积分规则的后续处理优先级。

## 当前阶段判断

- 还没有正式上线付费。
- 还没有第一个付费用户。
- 续费相关问题暂时不急。
- 中文用户付费概率低，中文支付后路径体验暂时不作为紧急项。
- Pro / Ultra 功能有一部分属于后续会加的能力，当前先不把它当成必须马上修的 blocker。

## P0 - 上线收款前必须定清楚

### 1. 一次性积分包购买策略

建议规则：

- 不要求先订阅才能买一次性积分包。
- 用户登录后即可购买一次性积分包。
- 订阅用户也可以买一次性积分包，用作月度积分不够时的补充。
- 游客不能购买，先登录再 checkout。

建议权限边界：

- 一次性积分包主要购买生成 / 导出额度。
- 一次性积分包默认解锁 Creator 级导出权益，例如无水印、商业使用。
- Pro / Ultra 高级能力仍绑定对应订阅，例如 Lip Sync、4K、高级模型、更低积分消耗、专业导出格式。
- 如果用户已有 Pro / Ultra 订阅，一次性积分应沿用当前订阅权限。

当前状态：

- 价格页已经展示 500 / 1,000 / 2,500 / 5,000 / 10,000 / 20,000 / 50,000 积分包和价格。
- 积分包 CTA 已接入 `/api/payment/checkout`，会创建 `type: "one-time"` 的 checkout。
- checkout payload 会传入 `credits`、`credits_valid_days: 365` 和美元价格。

后续仍需确认：

- 用真实 Stripe test checkout 验证支付成功后 credit ledger 是否正确增加积分。
- 确认一次性积分包是否应该永久提供 Creator 级导出权益，还是仅影响积分余额。

### 2. 年付积分发放规则

当前代码倾向：

- 月付 Creator 发 2,000 积分。
- 年付 Creator 会一次性发 24,000 积分。
- Pro / Ultra 年付同理按月额度乘以 12。

需要明确业务规则：

- 年付是一次性发全年积分，还是每月发一次？
- 年付积分是否到当前订阅周期结束时过期？
- 未用完积分是否滚存？

上线前要让三处一致：

- 价格页文案。
- checkout payload。
- 支付成功后的 credit ledger 发放和过期时间。

### 3. 积分估算器和真实扣费规则

当前风险：

- 价格页估算器可能高估用户需要的积分。
- 当前真实规则里，preview generation 约按音频秒数扣积分。
- 首轮图片生成包含在 preview generation 中，不额外扣图。
- 静态导出当前是 0 积分。
- 手动重新生成图片 / 单场景重试才按图片数量扣积分。

上线前建议：

- 让估算器只表达当前真实扣费。
- 或明确写成“粗略估算，不等于 checkout 或实际扣费规则”。

## P1 - 上线后较快处理

### 1. 套餐功能权限落地

当前接受现状：

- 页面上的 Pro / Ultra 能力很多会后续补上。
- 暂时不把这当成紧急修复。

后续仍需要补齐：

- Creator / Pro / Ultra 的功能边界要有后端鉴权。
- 不能长期只判断“有没有任意订阅”。
- 无水印、商业使用、高级模型、Lip Sync、4K、专业导出格式等能力要能按 plan 判断。

### 2. 一次性积分包与订阅权益的关系

需要实现：

- 非订阅用户购买积分包后，可以获得 Creator 级基础付费权益。
- 已订阅用户购买积分包后，积分消耗时沿用其当前订阅权益。
- 用户取消订阅后，剩余一次性积分是否仍保留 Creator 级权益，需要单独定规则。

## P2 - 暂时不急

### 1. 订阅续费补发积分

当前原因：

- 还没有付费用户，也没有真实续费周期。
- 续费问题只有用户进入下一次 billing cycle 时才会出现。

后续要修：

- Stripe `invoice.payment_succeeded` 需要区分 subscription create 和 renewal。
- renewal 应调用已有的 `handleSubscriptionRenewal()`，补发新周期积分并更新订阅周期。

### 2. 中文支付后路径

当前原因：

- 中文客户短期内不是主要付费入口。

后续要修：

- 从 `/zh/pricing` 发起 checkout 时，success / cancel URL 应保留 zh locale。
- 避免付款后跳到英文 `/settings/billing` 或 `/pricing`。

## 暂时不要做的事

- 不要因为还没接一次性积分包 checkout，就删除全部价格页设计。
- 不要为了快速上线，把 Pro / Ultra 所有未来能力都写成已经稳定可用。
- 不要先做复杂的套餐权限系统而阻塞第一个付费版本。
- 不要在没有业务规则前随意改年付积分发放逻辑。
